# 显示器亮度控制 UI

基于 DDC/CI 协议的显示器亮度控制程序，使用 Python tkinter 实现图形界面。  
支持从历史使用习惯中学习，启动时自动预测并调节亮度。

## 功能特性

- 支持多显示器选择
- 滑条调节亮度（0-100）
- 点击保存按钮应用亮度设置
- 查询当前显示器亮度
- 配置文件保存上次设置
- **智能亮度调节**：根据 history.log 中的历史记录，按时间段加权预测当前应有亮度，启动时自动应用
- 手动调节亮度会记录到 history.log，持续优化预测精度

## 智能亮度调节原理

### 算法流程

```
history.log
    │
    ▼
┌─────────────────────────────────────┐
│ 1. parse_history()                  │
│    正则提取 (时间戳, 亮度)           │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 2. dedup_records()                  │
│    10秒窗口内只取最后一条，          │
│    消除拖动滑条产生的中间值           │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 3. 按时分桶 + 时间加权               │
│    每条记录按距今时间指数衰减：       │
│    weight = 0.5^(天数/30)            │
│    半衰期 30 天                      │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 4. 桶级异常值防护（双重）            │
│    a) MIN_BUCKET_SIZE=3 过滤：       │
│       记录数 < 3 的桶直接丢弃        │
│    b) IQR 异常值剔除（≥5条时）：     │
│       Q1-1.5*IQR ~ Q3+1.5*IQR 外    │
│       的离群点被移除                  │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 5. 加权中位数                        │
│    weighted_median(values, weights)  │
│    对异常值免疫，比均值更鲁棒         │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 6. 线性插值 interpolate_brightness()│
│    在当前时间前后有效桶之间插值       │
│    · 支持跨午夜环形查找              │
│    · 5:00 分界规则：夜间(<5)与       │
│      白天(≥5)之间不插值，保持阶梯    │
└─────────────────────────────────────┘
    │
    ▼
  预测亮度值
```

### 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `DEDUP_WINDOW` | 10 秒 | 去重时间窗口 |
| `HALF_LIFE_DAYS` | 30 天 | 权重衰减半衰期 |
| `MIN_RECORDS` | 10 条 | 总记录数低于此值回退默认亮度 |
| `MIN_BUCKET_SIZE` | 3 条 | 小时桶记录数低于此值丢弃，不参与插值 |
| 默认亮度 | 50% | 数据不足时的回退值 |

### 异常值防护策略

由于历史记录来自用户手动操作，不可避免地存在异常值（如凌晨误触导致的高亮度记录）。算法采用三层防护：

1. **加权中位数**：替代加权均值，当桶内异常值为少数时，中位数几乎不受影响
2. **IQR 剔除**：对记录数 ≥ 5 的桶，计算四分位距 (Q1, Q3)，移除 1.5 倍 IQR 范围外的离群点，再算中位数
3. **最小桶容量**：记录数不足 3 条的桶直接丢弃，避免单条异常值主导整个桶

### 插值规则

- **正常时段**：在当前所在小时的前后两个有效桶之间线性插值
- **跨午夜**：支持环形查找（23时 → 0时）
- **5:00 分界**：夜间（5点前）与白天（5点后）之间不进行线性插值，直接取夜间值。这模拟了睡眠时段的亮度突变，避免在凌晨产生虚假的过渡亮度

> 自动调节不会记录到 history.log，只有手动调节才会被记录，确保历史数据始终反映真实使用偏好。

## 系统要求

- Windows 操作系统
- Python 3.8+
- 显示器支持 DDC/CI 协议

## 使用方法

### 使用 uv 运行

```bash
# 安装依赖
uv sync

# 运行程序
uv run python main.py

# 查看当前时间预测亮度
uv run python brightness_predictor.py
```

### 一键运行

双击 `run.bat` 文件即可启动程序，程序会自动检测系统时间并调节亮度。

## 文件说明

| 文件 | 说明 |
|------|------|
| `main.py` | 主程序入口，tkinter UI 界面 |
| `monitor_brightness.py` | DDC/CI 亮度控制底层模块 |
| `brightness_predictor.py` | 亮度预测模块，基于历史记录加权预测 |
| `analyze_history.py` | 历史数据分析脚本，按时间段统计亮度分布 |
| `config.json` | 配置文件（亮度、上次选择的显示器） |
| `history.log` | 亮度操作历史日志 |
| `run.bat` | 一键运行脚本 |

## 注意事项

- 需要显示器支持 DDC/CI 协议
- 某些显示器可能需要在 OSD 菜单中开启 DDC/CI 功能
- 可能需要管理员权限
- 首次使用时无历史数据，将使用默认亮度；积累 10 条以上手动记录后预测生效
